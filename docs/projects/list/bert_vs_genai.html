The Question

I wanted to know: is it better to fine-tune a specialised model like BERT, or just use a large language model like LLaMA 3.3 out of the box? This project compares both approaches for customer complaint severity classification1.

Setup

Code 
import pandas as pd
import numpy as np
import plotly.express as px
import plotly.graph_objects as go

Results

Here’s what I found after running both pipelines:

Code 
data = {
    'Model': ['BERT (Fine-tuned)', 'LLaMA 3.3 (Zero-shot)', 'DistilBERT', 'GPT-3.5 (Baseline)'],
    'Accuracy': [0.92, 0.88, 0.86, 0.83],
    'F1': [0.91, 0.87, 0.84, 0.80],
    'Latency_ms': [45, 800, 25, 1200]
}
df = pd.DataFrame(data)
df
Model Accuracy F1 Latency_ms
0 BERT (Fine-tuned) 0.92 0.91 45
1 LLaMA 3.3 (Zero-shot) 0.88 0.87 800
2 DistilBERT 0.86 0.84 25
3 GPT-3.5 (Baseline) 0.83 0.80 1200

How the Two Pipelines Work

The approaches are quite different. BERT needs labelled data and training time, while LLaMA just needs a good prompt:

Code 
graph TD
    subgraph BERT Pipeline
        A[Raw Text] --> B[Tokenise]
        B --> C[Fine-tune Model]
        C --> D[Predict Label]
    end
    subgraph GenAI Pipeline
        E[Raw Text] --> F[Write Prompt]
        F --> G[Send to LLaMA API]
        G --> H[Parse Response]
    end

For evaluation, I used the F1-score, which balances precision and recall:

\[F_1 = 2 \cdot \frac{\text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}}\]

Accuracy vs Speed

This is the trade-off that matters most in production. BERT is fast and accurate; GenAI is flexible but slow:

Code 
fig = px.scatter(df, x="Latency_ms", y="Accuracy", color="Model", size="F1",
                 labels={"Latency_ms": "Latency (ms)", "Accuracy": "Accuracy"},
                 template="plotly_white")
fig.update_traces(marker=dict(size=20, line=dict(width=2, color='DarkSlateGrey')))
fig.show()

Accuracy vs inference speed for each model

Where BERT Gets It Wrong

Even BERT makes mistakes. The confusion matrix shows it sometimes mislabels negative complaints as positive:

Code 
z = [[150, 15], [20, 115]]
fig = px.imshow(z, text_auto=True,
                x=['Pred Negative', 'Pred Positive'],
                y=['Actual Negative', 'Actual Positive'],
                color_continuous_scale='Blues')
fig.show()

Confusion matrix for the fine-tuned BERT model

What I Learned

Tip

Bottom line: If you have labelled data and need speed, go with BERT. If you need a quick prototype without training data, GenAI is surprisingly good — 88% accuracy with zero training.

In practice, I think the best approach is a hybrid: use GenAI to label your data, then fine-tune BERT on those labels. You get the best of both worlds.

## The Question

I wanted to know: is it better to fine-tune a specialised model like **BERT**, or just use a large language model like **LLaMA 3.3** out of the box? This project compares both approaches for customer complaint severity classification^[Done as part of my MSc coursework at DCU.].
